Turn a PR into a story you read outside-in — call-path chapters, explained, instead of alphabetical files.

ncr reorders a pull request so you meet the contract and entrypoint before the migration that serves them, each chapter explained in plain language with the code inline. A deterministic reconciler proves every changed line is shown — the model can't drop or alter a hunk.

Run it on your own PRs

ncr is a single Go binary. Point it at any GitHub PR and it builds the narrative, serves it locally, and opens your browser.

  1. 1

    Install

    brew install justinabrahms/tap/ncr

    Or go install github.com/justinabrahms/ncr@latest, or grab a prebuilt binary from Releases.

  2. 2

    Authenticate once

    ncr reads the PR through the GitHub CLI and calls Anthropic to plan the chapters.

    gh auth login
export ANTHROPIC_API_KEY=sk-ant-…
  3. 3

    Read a PR as a story

    ncr supabase/supabase 47266   # build, serve locally, open the browser
ncr owner/repo 812 --static   # write out/review.html and exit — no server
ncr .                         # self-review the current branch vs. its merge-base

    Every run is cached, so re-opening a PR is instant and free.
